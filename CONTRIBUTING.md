# Contributing to NBA Lab

Thank you for your interest in contributing to NBA Lab. This guide will help you get started with the development environment and workflow.

## Prerequisites

- Node.js 20+
- Python 3.12+
- uv (Python package manager)

## Getting Started

```bash
git clone <repo-url>
cd nba
npm install
uv sync
```

## Dual Toolchain

This project uses two distinct toolchains:

- **npm** manages the frontend (Next.js, React, TypeScript)
- **uv** manages the Python data pipeline (NetworkX, NumPy)

Both have lockfiles: `package-lock.json` and `uv.lock`. Always commit lockfile changes.

## Development

Start the development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

This performs a static export to `out/`.

## Code Quality

Pre-commit hooks are configured. Install them:

```bash
pre-commit install
```

Available commands:

- `npm run lint` — Lint JavaScript/TypeScript
- `npm run lint:py` — Lint Python
- `npm run format` — Format with Prettier
- `npm run format:py` — Format Python with Ruff
- `npm run typecheck` — TypeScript type checking

## Testing

Run tests for each part of the stack:

```bash
npm run test:js      # JavaScript tests
npm run test:py      # Python tests
npm run test:e2e     # E2E tests (requires Playwright)
```

Install Playwright browsers before running E2E tests:

```bash
npx playwright install
```

## Commit Guidelines

Use conventional commits with these prefixes:

- `feat:` — New features
- `fix:` — Bug fixes
- `docs:` — Documentation changes
- `test:` — Test additions or fixes
- `refactor:` — Code refactoring
- `chore:` — Maintenance tasks

Run `npm run lint && npm run typecheck` before committing.

## Generated Artifacts

Important notes about generated files:

- `out/` — Static build output (gitignored)
- `data/*.json` — Generated by Python pipeline (committed)
- `db/nba_raw_data.db` — Source database (committed)
- `*.egg-info/` — Python build artifacts (gitignored, do NOT commit)

## Project Structure

- Root `app/` contains Next.js layout plus a re-export to `src/app/page.tsx`
- Real page logic lives in `src/app/`
- Python scripts live in `scripts/`
